Catálogo simulado (data/catalog.json)

CLAUDE.md

Contexto del proyecto y decisiones de diseño documentadas.

Contexto

TechStore es una tienda online de tecnología. El catálogo real vive en una base de datos productiva; en este repo se simula con data/catalog.json para desarrollar y demostrar sin tocar producción.

Decisión 1

Catálogo simulado vía JSON

Qué: el MCP server y la skill leen catalog.json en vez de la DB productiva.

Por qué: reproducible por cualquiera que clone el repo, sin exponer credenciales reales.

Trade-off: no refleja stock en tiempo real.

Decisión 2

Umbral de stock bajo fijo

Qué: stock ≤ 5 unidades se marca "bajo", igual para todas las categorías.

Por qué: simplifica la v1 sin archivo de configuración que nadie mantendría.

Trade-off: una notebook con stock 5 alerta igual que un mouse con stock 5.

5 plantillas de prompt

Cada una probada con un caso real del catálogo. Desplegá el ejemplo para ver el output real.

1. Generador de descripción de producto

Al cargar un producto nuevo sin copy de venta
{nombre} · {specs} · {tono}
Ver ejemplo real ejecutado
Input
nombre=Monitor Samsung 24" FHD · specs=panel IPS, 75Hz, HDMI+VGA, VESA · tono=cercano y simple
Output
"Pasá tus horas frente a la pantalla a otro nivel. Este monitor Samsung de 24" con panel IPS te da colores fieles desde cualquier ángulo, y los 75Hz hacen que todo se vea más fluido..."

2. Reporte de stock bajo (versión prompt)

Lectura rápida del catálogo sin correr la skill
{catalogo_json} · {umbral}
Ver ejemplo real ejecutado
Input
catálogo completo (10 productos) · umbral=5
Output
TS-005 (stock 0), TS-007 (stock 1), TS-010 (stock 2), TS-003 (stock 3), TS-008 (stock 5) — ordenados de menor a mayor

3. Respuesta a ticket de soporte

Cliente pregunta por disponibilidad o precio
{producto} · {stock_actual} · {precio} · {nombre_cliente}
Ver ejemplo real ejecutado
Input
producto=SSD Kingston NV2 1TB · stock_actual=0 · precio=64.90 · cliente=Marcos
Output
"Hola Marcos, el SSD Kingston NV2 1TB tiene un precio de USD 64.90, pero por el momento está sin stock. Podemos anotarte para avisarte en cuanto vuelva a estar disponible..."

4. Changelog de catálogo

Después de actualizar precios/stock en lote
{cambios} · {fecha}
Ver ejemplo real ejecutado
Input
cambios=[TS-002 precio 17.99→19.99, TS-009 stock 12→40] · fecha=2026-06-15
Output
Precios: TS-002 (Mouse Logitech G203) 17.99 → 19.99 / Stock: TS-009 (Router TP-Link) 12 → 40

5. Clasificador de categoría

Producto nuevo sin categoría clara
{nombre_producto} · {categorias_existentes}
Ver ejemplo real ejecutado
Input
producto=Silla gamer ergonómica 4D · categorías=notebooks, perifericos, monitores...
Output
"Categoría nueva: mobiliario. Ninguna categoría existente cubre muebles..."

Skill: stock-report

Audita el catálogo y marca productos sin stock o con stock bajo (≤5 unidades). Corré el reporte real abajo.

MCP server: get_product_info

Consulta stock y precio por SKU exacto o nombre parcial. Probá con "TS-005", "monitor samsung" o "tostadora".

Ingresá un SKU o nombre para consultar el catálogo.
